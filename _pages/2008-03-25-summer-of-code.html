This is a design I made the Carcode project initiated by Dr.Toby Donaldson.

Catch up on this proj. I thought it was one cool way of initiation into the programming world.

Okay so this was only a basic design.

The project is some what in its infancy. There are still some cool features to be integrated, to realize sembelance of this design.

Some features the project will soon implement :

1> An interface to design your own cars.

It could be something like this. There can be provisions for uploading custom images of cars.

2> Next of course would be sensors fixed around the car. Pixel sensors are a mentioned future feature.

3> There should also be some kind of code-editor window:

Check out some of my posts on the Pyneko project. I've integrated a code-editor widget in program. I'm uploading a pic. here. For more details see http://gnuvision.com/books/pybook/

PS. Please download the earlier Pyneko version and all its dependencies before downloading the improved version. Replace the neko.py in the earlier versions with the neko-unni.py file which you can get as the improved version before you run it.

I thought the prospect of coding cars would be much more exciting than running a cat around,  (though the latter looks much more cute). Both Carcode and Pyneko targets novice Python learners. Though  PyNeko assumes nothing (being designed for school kids), Carcode targets a student community of the eleventh or twelvth grade with matbe some sort of programming base. The prospect of designing challenges for Carcode has a distinct thrill.
